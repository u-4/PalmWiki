はっきりfailしてくれる

fsによる操作が失敗した場合は常にerrorを返す。baseRはwarningとOS依存のエラーコードを返すことが多く、エラーを見逃しやすい。

fs::dir_ls("does_not_exist")
#> Error: [ENOENT] Failed to search directory 'does_not_exist': no such file or directory
dir("does_not_exist")
## character(0)

あ、これはかなり大きい。

どんな時でもUTF-8

baseRはOSのエンコードに依存する。

関数名がわかりやすい

  • path_*はpath関連の操作
  • file_*はファイル操作
  • dir_*はディレクトリの操作
  • link_*はリンクの操作
